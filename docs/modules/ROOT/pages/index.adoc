= Ultima Nexus JVM Check
:toc: left
:toclevels: 3

Centralized quality configuration for all projects in the Ultima Nexus JVM ecosystem.

== Philosophy

Ultima Nexus JVM Check enforces a *zero-tolerance* quality policy.
Every tool is configured with `maxErrors=0` and `maxWarnings=0` -- a single violation fails the build.
This ensures that code quality never degrades incrementally and that every contribution meets the same high standard from day one.

The rules are intentionally strict.
Rather than accumulating technical debt and retrofitting quality later, this project makes it impossible for violations to slip through.
When a rule is genuinely too restrictive for a specific case, the framework provides well-defined suppression mechanisms (see xref:customization.adoc[Customization]).

== What Is Included

The `ultima-nexus-jvm-check` artifact is a plain JAR that bundles configuration files for five quality tools:

[cols="1,2,1"]
|===
| Tool | Configuration File | Purpose

| Checkstyle
| `checkstyle.xml`
| Code style, Javadoc, naming, complexity

| PMD
| `pmdRuleset.xml`, `pmdRuleset-test.xml`
| Bug patterns, best practices, design

| SpotBugs
| `spotbugs-filter.xml`
| Bytecode-level bug detection exclusions

| Eclipse Formatter
| `java-formatter.xml`
| Deterministic auto-formatting (via Spotless)

| License Header
| `apache-license-header.txt`
| Apache 2.0 file header template
|===

All files are located at the root of the JAR classpath (`src/main/resources/`), so any Gradle or Maven plugin can reference them once the artifact is on the build classpath.

== Coordinates

[source,xml,subs="+attributes"]
----
<dependency>
    <groupId>dev.daymor</groupId>
    <artifactId>ultima-nexus-jvm-check</artifactId>
    <version>1.0.0</version>
</dependency>
----

== Quick Setup

=== Gradle (Kotlin DSL)

Add the dependency so that quality plugins can resolve the configuration files from the classpath.

[source,kotlin]
----
// settings.gradle.kts
dependencyResolutionManagement {
    repositories {
        mavenCentral()
    }
}
----

[source,kotlin]
----
// build.gradle.kts
plugins {
    java
    checkstyle
    pmd
}

val qualityConfig: Configuration by configurations.creating

dependencies {
    qualityConfig("dev.daymor:ultima-nexus-jvm-check:1.0.0")
}

// Extract the config JAR so tools can read the files
val extractQualityConfig by tasks.registering(Copy::class) {
    from(qualityConfig.map { zipTree(it) })
    into(layout.buildDirectory.dir("quality-config"))
}

checkstyle {
    toolVersion = "10.23.1"
    maxErrors = 0
    maxWarnings = 0
    configFile = layout.buildDirectory.file("quality-config/checkstyle.xml").get().asFile
}

tasks.withType<Checkstyle>().configureEach {
    dependsOn(extractQualityConfig)
}

pmd {
    toolVersion = "7.17.0"
    isConsoleOutput = true
    ruleSetConfig = resources.text.fromFile(
        layout.buildDirectory.file("quality-config/pmdRuleset.xml")
    )
    ruleSets = emptyList()
}

tasks.withType<Pmd>().configureEach {
    dependsOn(extractQualityConfig)
}
----

TIP: If you use the Ultima Nexus JVM Gradle plugins, this wiring is handled automatically -- you do not need to write any of the above.

=== Maven

[source,xml]
----
<project>
    <build>
        <plugins>
            <plugin>
                <groupId>org.apache.maven.plugins</groupId>
                <artifactId>maven-checkstyle-plugin</artifactId>
                <version>3.6.0</version>
                <dependencies>
                    <dependency>
                        <groupId>dev.daymor</groupId>
                        <artifactId>ultima-nexus-jvm-check</artifactId>
                        <version>1.0.0</version>
                    </dependency>
                </dependencies>
                <configuration>
                    <configLocation>checkstyle.xml</configLocation>
                    <maxAllowedViolations>0</maxAllowedViolations>
                </configuration>
            </plugin>
        </plugins>
    </build>
</project>
----

Because the configuration files sit at the classpath root, Maven plugins resolve them by name alone.

== Tool Versions

The configuration files are tested against the following tool versions.
Using significantly older versions may cause parse errors or missing-rule warnings.

[cols="1,1"]
|===
| Tool | Tested Version

| Checkstyle
| 10.23.1+

| PMD
| 7.17.0+

| SpotBugs
| 4.9.x+

| Eclipse JDT Formatter
| 4.34+ (profile version 23)
|===
