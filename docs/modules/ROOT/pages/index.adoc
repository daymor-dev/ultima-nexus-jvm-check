= Ultima Nexus JVM Check
:toc: left
:toclevels: 3

Centralized quality configuration for all projects in the Ultima Nexus JVM ecosystem.

== Philosophy

Ultima Nexus JVM Check enforces a *zero-tolerance* quality policy.
Every tool is configured with `maxErrors=0` and `maxWarnings=0` -- a single violation fails the build.
This ensures that code quality never degrades incrementally and that every contribution meets the same high standard from day one.

The rules are intentionally strict.
Rather than accumulating technical debt and retrofitting quality later, this project makes it impossible for violations to slip through.
When a rule is genuinely too restrictive for a specific case, the framework provides well-defined suppression mechanisms (see xref:customization.adoc[Customization]).

== What Is Included

The `ultima-nexus-jvm-check` artifact is a plain JAR that bundles configuration files for five quality tools:

[cols="1,2,1"]
|===
| Tool | Configuration File | Purpose

| Checkstyle
| `checkstyle.xml`
| Code style, Javadoc, naming, complexity

| PMD
| `pmdRuleset.xml`, `pmdRuleset-test.xml`
| Bug patterns, best practices, design

| SpotBugs
| `spotbugs-filter.xml`
| Bytecode-level bug detection exclusions

| Eclipse Formatter
| `java-formatter.xml`
| Deterministic auto-formatting (via Spotless)

| License Header
| `apache-license-header.txt`
| Apache 2.0 file header template
|===

All files are located at the root of the JAR classpath (`src/main/resources/`), so any Gradle or Maven plugin can reference them once the artifact is on the build classpath.

== Coordinates

[source,xml,subs="+attributes"]
----
<dependency>
    <groupId>dev.daymor.ultima-nexus.jvm</groupId>
    <artifactId>ultima-nexus-jvm-check</artifactId>
    <version>1.0.0</version>
</dependency>
----

== Quick Setup

=== With Ultima Nexus Gradle Plugins (recommended)

If you use the xref:ultima-nexus-jvm-gradle-plugins::index.adoc[Ultima Nexus Gradle Plugins], all quality tools are wired automatically — no manual configuration needed:

[source,kotlin]
----
// settings.gradle.kts
plugins {
    id("dev.daymor.ultimanexus.jvm.gradle.bundle.ultima-nexus-jvm-settings") version "1.0.0"
}
----

[source,kotlin]
----
// build.gradle.kts
plugins {
    alias(libs.plugins.ultimanexus.jvm.bundle.spring.boot.complete.application)
}
----

Run `./gradlew qualityCheck` to verify. All Checkstyle, PMD, SpotBugs, and formatting rules from this project are applied automatically.

=== Standalone (without Ultima Nexus Gradle Plugins)

Add the artifact to your build classpath and point each tool to its configuration file:

.Gradle (Kotlin DSL)
[source,kotlin]
----
dependencies {
    checkstyle("dev.daymor.ultima-nexus.jvm:ultima-nexus-jvm-check:1.0.0")
}

checkstyle {
    maxErrors = 0
    maxWarnings = 0
    configFile = resources.text.fromArchiveEntry(
        configurations.checkstyle.get(), "checkstyle.xml"
    ).asFile()
}
----

.Maven
[source,xml]
----
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-checkstyle-plugin</artifactId>
    <dependencies>
        <dependency>
            <groupId>dev.daymor.ultima-nexus.jvm</groupId>
            <artifactId>ultima-nexus-jvm-check</artifactId>
            <version>1.0.0</version>
        </dependency>
    </dependencies>
    <configuration>
        <configLocation>checkstyle.xml</configLocation>
        <maxAllowedViolations>0</maxAllowedViolations>
    </configuration>
</plugin>
----

Because the configuration files sit at the classpath root, tools resolve them by name alone.

== Tool Versions

The configuration files are tested against the following minimum tool versions.
Using significantly older versions may cause parse errors or missing-rule warnings.
The actual tool versions are set by the Gradle plugins (or your own build configuration), not by this project.

[cols="1,1"]
|===
| Tool | Tested Version

| Checkstyle
| 10.23.1+

| PMD
| 7.17.0+

| SpotBugs
| 4.9.x+

| Eclipse JDT Formatter
| 4.34+ (profile version 23)
|===
