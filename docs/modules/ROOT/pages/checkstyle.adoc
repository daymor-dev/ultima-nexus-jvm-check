= Checkstyle
:description: Zero-tolerance Checkstyle configuration for Ultima Nexus JVM

The project enforces Checkstyle with `maxErrors=0` and `maxWarnings=0`, meaning every violation is a build failure.
The severity label (`error` vs `warning`) distinguishes structural defects from style preferences in reports, but both block the build.
The Checkstyle version is configured by the Gradle plugin, not by this project.

== Philosophy

Code style is not optional.
Automated formatting (Spotless) handles whitespace and import ordering; Checkstyle catches everything else -- naming, Javadoc, complexity, and correctness patterns.
By enforcing zero warnings the codebase stays consistent without "fix later" debt.

== Suppressions

Two optional suppression mechanisms are supported:

`${checkstyle.file.suppressions}`::
A file-level `SuppressionFilter` (XPath-free).
Accepts a file path via a Gradle/Maven property.
Used to suppress entire files or file patterns.

`${checkstyle.suppressions}`::
A TreeWalker-level `SuppressionXpathFilter`.
Accepts a file path via a Gradle/Maven property.
Used for fine-grained, XPath-based suppressions on specific AST nodes.

Both properties have `optional="true"`, so builds succeed when no suppression file is provided.

See xref:customization.adoc[Customization] for details on writing suppression files.

== Custom Rules

=== No Boolean Parameters

Public methods in production code must not accept `boolean` parameters -- use enums or dedicated types instead.

Exemptions: fluent setters (single `boolean` parameter assigning to a field), fluent builder-style setters returning `this`, and test classes (`*Test`, `*IT`, `*FT`, `*PT`).

=== BDDMockito Enforcement

A `RegexpSinglelineJava` check forbids `org.mockito.Mockito.when`, `doThrow`, and `doAnswer` in favor of BDDMockito equivalents (`given`, `willThrow`, `willAnswer`).

=== AssertJ Enforcement

A `RegexpSinglelineJava` check forbids `org.junit.Assert.assert*` in favor of AssertJ assertions.

== Built-in Suppressions

The configuration includes several XPath-based suppressions for common framework patterns:

Fluent setters::
`HiddenField` is suppressed for methods that take a single parameter, assign it to a field via `this.field = param`, and return `void` or `this`.

`@SpringBootApplication` classes::
`HideUtilityClassConstructor` is suppressed because Spring Boot requires a public constructor.

Lombok `@UtilityClass`::
`AbbreviationAsWordInName` and `MemberName` are suppressed on `final` fields (Lombok makes them `static final`).
A `MatchXpath` rule re-enforces `UPPER_SNAKE_CASE` for those constants.

Test classes::
`IllegalThrows` is suppressed for classes whose names end in `Test`, `IT`, `FT`, or `PT`, since test methods commonly declare `throws Exception`.
